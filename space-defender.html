Project Overview

Space Defender is a browser-based space shooter game built entirely with vanilla JavaScript and WebGL. The goal was to create a performant, engaging game with modern visual effects that runs smoothly at 60 FPS without relying on game engines or heavy frameworks.

Challenge

Managing real-time physics, collision detection, particle systems, and enemy AI while maintaining 60 FPS performance across all devices.

Solution

Implemented efficient object pooling, spatial partitioning for collision detection, and WebGL-based rendering for particle effects.

Game Loop Architecture

// Main game loop
class SpaceGame {
    constructor() {
        this.lastTime = 0;
        this.fps = 60;
        this.gameState = 'menu';
    }

    // 60 FPS game loop
    gameLoop(currentTime) {
        const deltaTime = currentTime - this.lastTime;
        this.lastTime = currentTime;
        
        // Calculate FPS
        this.fps = 1000 / deltaTime;
        
        // Update game state
        this.update(deltaTime);
        
        // Render frame
        this.draw();
        
        // Continue loop
        requestAnimationFrame((time) => this.gameLoop(time));
    }
}

Optimization Techniques

  • Object Pooling: Reusing game objects instead of creating/destroying
  • Spatial Partitioning: Grid-based collision detection optimization
  • Canvas Layer Separation: Separating static and dynamic elements
  • Debounced Rendering: Only rendering when necessary
  • Efficient Particle System: Using linked lists for particle management

Results & Metrics

Mobile Performance

Maintained 60 FPS on mid-range mobile devices through careful optimization and touch-friendly controls.

User Engagement

Average play session of 8 minutes with 75% of players reaching level 3 or higher.

The game successfully demonstrates that complex, visually rich games can be built with vanilla JavaScript while maintaining professional-grade performance and polish.

Lessons Learned

This project reinforced the importance of performance optimization in real-time applications. Key takeaways include:

  • Object pooling is essential for garbage collection management in games
  • Spatial partitioning dramatically improves collision detection performance
  • Canvas context state changes are expensive and should be minimized
  • Mobile touch controls require careful UX consideration
  • Progressive difficulty keeps players engaged longer
